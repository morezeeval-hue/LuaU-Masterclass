--!strict
export type Item = {
    id: string,
    name: string,
    amount: number,
}

local Inventory = {}
Inventory.__index = Inventory

function Inventory.new()
    return setmetatable({ items = {} :: {[string]: Item} }, Inventory)
end

function Inventory:Add(item: Item)
    local existing = self.items[item.id]
    if existing then
        existing.amount += item.amount
    else
        self.items[item.id] = item
    end
end

return Inventory

Roblox Studio und Engine-Konzepte

Studio

Explorer zeigt das Datenmodell. Properties verändert Instanzwerte. Output zeigt Prints, Warnungen und Errors. Command Bar testet kleine Snippets. Asset Manager verwaltet Bilder, Meshes, Sounds und Packages. Device Emulator prüft UI auf Mobile, Tablet und Desktop.

Wichtige Container: Workspace für 3D-Welt, ReplicatedStorage für geteilte Module/Remotes, ServerStorage für serverseitige Assets, ServerScriptService für Servercode, StarterGui für UI, StarterPlayerScripts für Clientcode.

Replikation

Der Server ist die Quelle der Wahrheit. Clients dürfen Eingaben melden, aber keine endgültigen Spielwerte bestimmen. Alles, was exploitable ist, muss auf dem Server geprüft werden: Geld, Schaden, Items, Positionen, Cooldowns, Käufe und Belohnungen.

RemoteEvent ist für einseitige Nachrichten. RemoteFunction blockiert bis zur Antwort und sollte sparsam verwendet werden. Für interne Server- oder Client-Kommunikation nutzt man Bindables oder eigene Signals.

Sicheres RemoteEvent-Pattern

Der Client fragt an. Der Server validiert. Nur der Server verändert echte Spielwerte.

-- ServerScriptService/Purchase.server.lua
local ReplicatedStorage = game:GetService("ReplicatedStorage")
local Players = game:GetService("Players")

local PurchaseItem = ReplicatedStorage.Remotes.PurchaseItem
local PRICES = { Sword = 100, Potion = 25 }
local cooldown = {}

PurchaseItem.OnServerEvent:Connect(function(player, itemId)
    if typeof(itemId) ~= "string" then return end
    if not PRICES[itemId] then return end

    local now = os.clock()
    if cooldown[player] and now - cooldown[player] < 0.5 then return end
    cooldown[player] = now

    local leaderstats = player:FindFirstChild("leaderstats")
    local coins = leaderstats and leaderstats:FindFirstChild("Coins")
    if not coins or coins.Value < PRICES[itemId] then return end

    coins.Value -= PRICES[itemId]
    -- Item serverseitig vergeben, dann UI-Update an Client senden.
end)

Players.PlayerRemoving:Connect(function(player)
    cooldown[player] = nil
end)

Qualität, Tools und Release

Debugging

Nutze Output, Breakpoints, Watches, Script Analysis, server/client getrennte Tests, gezielte Warnungen, Assertions und reproduzierbare Testfälle. Prüfe immer: läuft der Code auf Server oder Client, existiert die Instanz schon, repliziert sie überhaupt?

Performance

Vermeide unnötige Schleifen pro Frame, zu viele Instances, unkontrollierte Connections, große Remote-Payloads und permanente Polling-Logik. Nutze Events statt Dauerschleifen, räume Connections auf und profile echte Szenen.

Team-Workflow

Rojo für Dateisystem-Workflows, Git für Versionierung, Wally für Packages, Aftman für Tool-Versionen, Selene/Stylua für Codequalität, TestEZ für Tests. Kleine Module, klare APIs, Code Reviews.

Release

Teste mit mehreren Spielern, Mobile und Controller. Prüfe DataStore-Fehler, Exploit-Flows, Performance auf schwächeren Geräten, Kaufverarbeitung, private Server, Teleports und Rollback-Plan.
